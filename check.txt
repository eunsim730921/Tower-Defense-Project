나는 소프트웨어 학과 3학년이고, 기존 웹 게임에 기능을 추가하는 과제를 하고있어 
내가 올린 파일은 기존 웹 게임 코드이고 협업 방식으로 진행 중이야 
나는 타워 관련
    (타워 종류, 
     타워 사거리, 
     타워 공격력, 
     타워 공격속도,

2. [타워에 마우스 올리거나 클릭 시 사거리 표시]
타워를 클릭하거나 마우스를 올려둘 때(hover) 사거리가 표시되게 해줘. 클릭을 했을 때는 다시 클릭을 하기 전까지 사거리 표시가 유지돼야해

3. [맵 격자 형태로 변경]
맵 형태를 격자형태로 만들어서 모든 부분에 타워를 설치할 수 있게 해줘. 
격자 형태는 시각적으로 볼 수 있어야하고, 적 이동경로에는 타워를 설치할 수 없어

4. [tower_defense.js , tower.js 기능별 별도 js 파일 분리]
tower_defense.js와 tower.js를 기능별로 별도의 js 파일로 분리해줘. 
각 파일은 한 기능에만 집중하도록 만들고, import/export 없이 전역 변수로 상호작용하는 구조로 만들어줘. 
대신 tower_defense.js와 tower.js의 기능이 파일을 변경했음에도 변함없이 똑같이 작동하고 
file:// 환경에서도 바로 실행 가능해야 해 . 
    ex) main.js / map.js / enemy.js 등등

5. [타워 종류 추가]
Have a Break, and then.. 먼저 이 질문에 답하기 위한 최적의 접근 전략을 설계하고, 
그 전략에 따라 체계적으로 문제를 해결해줘 
[질문] 현재 격자 형태로 된 맵의 타일을 클릭하면 바로 타워가 생성 돼. 
이 타일을 누르면 타일이 연한 파란색(선택된 타일이라는 뜻)으로 바뀌고,
id="towerInfoPanel" 여기에 타워 종류가 뜨고 원하는 타워 종류를 선택하면 해당 타워를 타일에 배치하게 만들어줘

6. [타워팔기 기능 추가]
타워를 판매할 수 있는 기능을 추가해줘. 
각 타워가 건설 및 강화에 사용한 모든 금액을 하나의 변수(investedMoney)에 저장하도록 하고, 
팔기 버튼을 누르면 해당 타워의 투자 금액의 70%를 환급받도록 만들어줘. 
기존 tower.js의 다른 기능(타워 생성, 강화, 공격, 사거리 표시 등)은 변경 없이 그대로 유지되어야 해.
